Dashboard / Manage Sections

Homepage & Portfolio case studies

How this works

Both the homepage's "Selected work" section and the full Portfolio.dc.html page can pull their case-study cards live from real posts in /posts/. Load the post files below, tick Homepage and/or Portfolio for each one, set the order, then generate the config file(s) and drop them into the project root (next to index.html). This is still a static site, so there's no live save — each page re-reads its config file on load, and falls back to whatever cards are already hand-written on that page if the file is missing.

Checking what's currently live…

Loaded automatically so a new post gets added to the existing lineup instead of replacing it. If this site isn't deployed yet, or you're testing locally, this will just find nothing — that's fine.

1 · Load a new post

Click to select one or more .html files from your /posts/ folder, or drag them here.

Nothing is uploaded anywhere — these files are only read in your browser to pull out the cover image, title, sector, year and summary.

2 · Choose & order

Tick Homepage and/or Portfolio for each post. Use the arrows to set the order — on the homepage only the first two Homepage-ticked posts show (it's a 2-slot grid); Portfolio shows every Portfolio-ticked post.

Downloads featured.json (if any post is ticked Homepage) and/or portfolio.json (if any post is ticked Portfolio). Save whichever ones download into the project root, alongside index.html.
